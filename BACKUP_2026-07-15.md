## 2. Base de Datos Supabase (Esquema y Tablas)
Se ha diseñado el esquema del módulo de sanciones internas para los futbolistas de la plantilla:

### Tablas del Módulo de Sanciones

#### `player_fines`
Registra el historial individual de multas de cada jugador de la plantilla.
- **Campos**:
  - `id` (UUID, PK, Default: `gen_random_uuid()`)
  - `player_id` (UUID REFERENCES `players(id)` ON DELETE CASCADE, NOT NULL)
  - `motivo` (TEXT, NOT NULL)
  - `fecha` (DATE, NOT NULL)
  - `contexto` (TEXT CHECK `contexto IN ('Entrenamiento', 'Partido', 'Otro')`, NOT NULL)
  - `evento_id` (UUID, NULL) - Referencia opcional al ID de la tabla `matches` o `planning_sessions`
  - `evento_nombre` (TEXT, NULL) - Nombre del partido/sesión o texto libre descriptivo si el contexto es 'Otro'
  - `importe` (NUMERIC(10, 2), NOT NULL)
  - `cantidad` (INTEGER, Default: 1, NOT NULL)
  - `estado` (TEXT CHECK `estado IN ('Pendiente', 'Pagado')`, Default: 'Pendiente', NOT NULL)
  - `observaciones` (TEXT, NULL)
  - `created_at` (TIMESTAMPTZ, Default: `now()`, NOT NULL)
  - `updated_at` (TIMESTAMPTZ, Default: `now()`, NOT NULL)

### Políticas RLS (Row Level Security)
- Habilitada en la tabla `player_fines`.
- Política de lectura libre de base de datos para alineación con las tablas de lesiones y reuniones:
  - `CREATE POLICY "Public SELECT only" ON public.player_fines FOR SELECT USING (true);`
- Operaciones de escritura (INSERT/UPDATE/DELETE) denegadas para SELECT público y controladas estrictamente a través de los RPC de staff existentes (`exec_secure_upsert` y `exec_secure_delete`).

## 3. Código y Hook Creado
- **React Hook (`hooks/usePlayerFines.ts`)**: Encapsula las lecturas directas por SELECT e inserta/modifica/elimina registros utilizando las llamadas RPC seguras enviando la clave del staff y comprobando permisos locales de edición mediante `verifyWritePermission()`.
- **Vista de UI (`components/players/PlayerDetail.tsx`)**: Integración del panel de visualización, filtros rápidos por estado y ámbito, indicadores de sumatorios financieros y el modal formulario adaptativo con multiplicadores dinámicos.
